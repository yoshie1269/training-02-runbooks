# ストレージ管理 手順書
## 既存のEC２にEBSを追加する

## 1. Amazon EBS ボリュームの作成

### 1-1. AWS マネジメントコンソールにログイン
- AWS マネジメントコンソールにログインする
- **EC2** サービスを開く

---

### 1-2. ボリューム作成画面を開く
- 左側ナビゲーションペイン（nav）で  
  **[Elastic Block Store] → [ボリューム]** を選択
- **[ボリュームの作成]** を選択する

---

### 1-3. ボリューム設定
以下の項目を設定する

- **ボリュームタイプ**  
  - 作成するボリュームのタイプを選択  
  - 例：`gp3`

- **サイズ**  
  - ボリュームのサイズ（GiB）を入力  
  - 例：`10 GiB`

- **アベイラビリティーゾーン**  
  - ボリュームを作成する AZ を選択  
  - ※ アタッチする EC2 と **同じ AZ** にする必要がある

---

### アベイラビリティゾーン確認方法

EC2 → インスタンス → 対象インスタンス → サブネットID

---

- **スナップショットID**  
  - デフォルト値のまま（空欄）

---

### 1-4. タグ設定
- **[タグの追加]** を選択する
- 以下のタグを設定する

| キー | 値 |
|---|---|
| Name | 任意（例：additional-ebs-01） |

---

### 1-5. ボリューム作成
- **[ボリュームの作成]** を選択する
- ボリュームのステータスが **available** になっていることを確認する

---

## 2. Amazon EBS ボリュームのアタッチ

### 2-1. ボリューム一覧を開く
- EC2 コンソールを開く
- **[Elastic Block Store] → [ボリューム]** を選択

---

### 2-2. ボリュームをアタッチする
- アタッチ対象のボリュームを選択する
- **[アクション] → [ボリュームのアタッチ]** をクリックする

---

### 2-3. アタッチ設定
- **インスタンス**  
  - アタッチ先の EC2 インスタンスを選択する

  - **デバイス名**  
  - リストからデバイス名を選択する  
  - 例：`/dev/xvdf`

  > ※ OS 上では `/dev/nvme1n1` などの名前で認識される場合がある

- **[アタッチ]** を選択する

---

### 2-4. アタッチ確認
- ボリュームの状態が **in-use** になっていることを確認する

---

## 3. Amazon EBS ボリュームを使用できるようにする

### 3-1. EC2 インスタンスへ接続
- ターミナルを開く
- SSH を使用して EC2 インスタンスに接続する

```bash
ssh -i key.pem ec2-user@<パブリックIP>
```

---

### 3-2. デバイス確認（lsblk）

使用可能なディスクデバイスとマウントポイントを確認する。

```bash
lsblk
```
#### lsblk コマンドについて

- ストレージをツリー構造で表示するコマンド
- 以下の目的で使用する  
  - EBS 追加前後のディスク構成の確認  
  - 使用する正しいデバイス名の特定  
  - マウント状況の確認

#### 出力項目の意味

| 項目 | 説明 |
|---|---|
| NAME | デバイス名（xvda, xvdf, nvme1n1 など） |
| MAJ:MIN | メジャー番号:マイナー番号（管理用） |
| RM | リムーバブルか（1=Yes / 0=No） |
| SIZE | デバイスの容量 |
| RO | 読み取り専用か（1=Yes / 0=No） |
| TYPE | 種類（disk / part など） |
| MOUNTPOINT | マウント先ディレクトリ |

---

### 3-3. ファイルシステムの作成

新規で作成した Amazon EBS ボリュームにファイルシステムを作成する。

> ※ 新規で作成した EBS ボリュームのみ実施する  
> ※ 既存データがあるボリュームに対しては実行しないこと

```bash
sudo mkfs -t xfs /dev/xvdf
```

- `/dev/xvdf` は `lsblk` コマンドで確認したデバイス名に置き換える

---

### 3-4. マウントポイントの作成

EBS ボリュームをマウントするためのディレクトリを作成する。

```bash
sudo mkdir /data
```
---

### 3-5. ボリュームのマウント

作成したマウントポイントに EBS ボリュームをマウントする。

```bash
sudo mount /dev/xvdf /data
```
---

### 3-6. マウント確認

EBS ボリュームが正しくマウントされていることを確認する。

```bash
lsblk
df -h
```

- `/data` に EBS ボリュームがマウントされていることを確認する

---

### 3-7. デバイスの UUID を確認する

自動マウント設定では、デバイス名ではなく **UUID** を使用する。  
UUID を確認するため、以下のコマンドを実行する。

```bash
sudo blkid
```

#### blkid コマンドについて
- ブロックデバイスの UUID やファイルシステム種別を表示する
- `/etc/fstab` 設定時に必要な情報を取得するために使用する

#### 出力例（一部）

```bash
/dev/xvdf: UUID="1234abcd-5678-efgh-9012-ijklmnopqrst" TYPE="xfs"
```

- この手順書では、以下を使用する  
  - UUID：`1234abcd-5678-efgh-9012-ijklmnopqrst`
  - ファイルシステム：`xfs`

---

### 3-8. /etc/fstab のバックアップを作成する

設定ミスに備えて、事前に `/etc/fstab` のバックアップを作成する。

```bash
sudo cp /etc/fstab /etc/fstab.bak
```
---

### 3-9. /etc/fstab を編集する

`/etc/fstab` ファイルを編集し、自動マウント設定を追加する。

```bash
sudo vi /etc/fstab
```

#### 追記内容（例）

```bash
UUID=1234abcd-5678-efgh-9012-ijklmnopqrst /data xfs defaults,nofail 0 2
```

#### 各項目の意味

| 項目 | 説明 |
|---|---|
| UUID | マウントするデバイスの識別子 |
| /data | マウントポイント |
| xfs | ファイルシステム種別 |
| defaults,nofail | マウントオプション |
| 0 | dump の対象かどうか |
| 2 | fsck の実行順序 |

- `nofail` を指定することで、EBS が存在しない場合でも OS 起動が停止しない

---

### 3-10. 設定内容の確認（再マウント）

`/etc/fstab` の設定が正しいかを確認する。

```bash
sudo umount /data
sudo mount -a
```

---

### 3-11. 自動マウントの確認

再度マウント状況を確認する。

```bash
lsblk
df -h
```

- `/data` が表示されていれば、自動マウント設定は正常

---

### 3-12. 再起動後の確認

インスタンスを再起動し、再起動後も自動でマウントされていることを確認する。

```bash
sudo reboot
```

再起動後、再度以下を確認する。

```bash
lsblk
df -h
```

---

## 4. まとめ

1. Amazon EBS ボリュームを作成する  
2. EC2 インスタンスにアタッチする  
3. OS 上でデバイスを確認する  
4. ファイルシステムを作成する  
5. マウントして使用可能にする  
6. `/etc/fstab` を設定し、自動マウントを有効にする  

---

## 注意事項
- `/etc/fstab` の記述を誤ると、OS が起動しなくなる可能性がある  
- 必ずバックアップを取得してから編集する  
- UUID を使用することで、デバイス名変更による影響を防ぐ
