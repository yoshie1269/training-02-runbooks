# ■ 1. Router 初期設定

## 1-1. IPアドレス設定

### この工程でしていること
- ルータのインターフェースにIPアドレスを設定する
- L3通信できる状態にする
- shutdown状態のIFを有効化する


### 実行コマンド
```bash
conf t
interface gigabitEthernet 0/0/0
ip address <IPアドレス> <ネットマスク>
no shutdown
exit
end
```


### コマンドの意味

| コマンド | 意味 |
|-----------|------|
| conf t | グローバル設定モード |
| interface | 設定対象IF選択 |
| ip address | IP設定 |
| no shutdown | IF有効化 |
| end | 設定終了 |


### 確認
```bash
show ip interface brief
```

### OKの目安
- 対象IFが `up/up`
- IPアドレスが正しく表示される


## 1-2. 設定の保存

### この工程でしていること
- 現在の設定（RAM）を再起動後も残るよう保存する


### 実行コマンド
```bash
copy running-config startup-config
```
### 意味
- running-config（現在設定）をstartup-config（保存設定）へコピー

# ■ 2. ルーティング設定

# 2-1. スタティックルート


### この工程でしていること
- 手動で経路を登録する
- 特定ネットワークへの転送先を明示する


### 実行コマンド
```bash
ip route <宛先ネットワーク> <ネットマスク> <ネクストホップIP>
```

### 例
```bash
ip route 192.168.2.0 255.255.255.0 10.0.0.2
```

### 確認
```bash
show ip route
```

### OKの目安
```bash
S 192.168.2.0/24 [1/0] via 10.0.0.2

S = Static
```

# 2-2. ダイナミックルート（OSPF）


### この工程でしていること
- OSPFを有効化
- 隣接ルータと経路交換する


### 実行コマンド
```bash
router ospf 1
network <IPアドレス> <ワイルドカードマスク> area <エリア番号>
end
```

### ワイルドカードマスク例
| サブネット | ワイルドカード |
|------------|---------------|
| /24 | 0.0.0.255 |
| /30 | 0.0.0.3 |


### 確認
```bash
show ip ospf neighbor
show ip route
```

### OKの目安
- Neighborが表示される
- O（OSPF）経路が表示される


# 2-3. アドミニストレーティブディスタンス（AD）


### この工程でしていること
- 経路の優先度を制御する


### 実行コマンド
```bash
ip route <宛先ネットワーク> <ネットマスク> <ネクストホップ> <AD値>
```


### ポイント
| ルート種別 | AD |
|------------|----|
| Static | 1 |
| OSPF | 110 |

→ 数値が小さいほど優先

# ■ 3. FHRP（HSRP）設定

## 構成

仮想IP：
- 172.31.0.5（sw00側）
- 172.31.1.5（sw01側）


## この工程でしていること
- 複数ルータで仮想IPを共有
- デフォルトゲートウェイを冗長化


## 設定手順
```bash
conf t
interface gigabitEthernet 0/0/0
standby <グループID> ip <仮想IPアドレス>
exit
```

## 優先度設定（推奨）
```bash
standby <グループID> priority 110
standby <グループID> preempt
```


## 確認
```bash
show standby brief
```

### OKの目安
- Active / Standby が正しく表示される

<br>

# ■ 4. リンクアグリゲーション（EtherChannel）


## 構成

pc0：172.31.0.1/24  
pc1：172.31.0.2/24  

sw2：  
- fa0/1 → pc0  
- fa0/2-3 → sw3  

sw3：  
- fa0/1 → pc1  
- fa0/2-3 → sw2  

## この工程でしていること
- 複数物理リンクを1本の論理リンクとして束ねる
- 帯域向上＋冗長化
- STPによるブロック回避


## 設定手順（sw2 / sw3 両方で実施）


### 1. VLAN作成
```bash
vlan <任意番号>
exit
```


### 2. EtherChannel設定
```bash
interface range <IF名>, <IF名>
channel-group <番号> mode active
no shutdown
exit
```

### コマンドの意味

| コマンド | 意味 |
|-----------|------|
| interface range | 複数IF同時設定 |
| channel-group | ポート束ね |
| mode active | LACP有効 |

---

### 3. ポートチャネル設定
```bash
interface port-channel 1
switchport mode access
switchport access vlan 2
```

## 確認
```bash
show etherchannel summary
show spanning-tree
```


### OKの目安
- Port-channelがSU状態
- STPでブロックされない

# ■ 学習まとめ

・IP設定 → IFをupにするのを忘れない  
・Staticは明示的経路  
・OSPFは動的経路  
・ADで優先制御可能  
・HSRPは仮想IP共有によるGW冗長化  
・EtherChannelは複数回線束ねて帯域＋冗長
