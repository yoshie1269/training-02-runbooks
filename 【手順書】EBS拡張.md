# ストレージ管理 手順書  
## EBS ボリュームサイズ変更後の Linux ファイルシステム拡張

---

## 概要

AWS マネジメントコンソールで EBS ボリュームのサイズを変更した後、  
Linux OS 上で **拡張された容量を認識させ、ファイルシステムを拡張して使用可能にする**ための手順を示す。

---

## 1. 前提条件
- EBS ボリュームのサイズ変更が完了していること  
- ボリュームのステータスが **optimizing** または **completed** であること  
- 対象ボリュームが EC2 インスタンスにアタッチされていること  
- ファイルシステムがすでに作成済みであること  

---

## 2. EC2 インスタンスへ接続

ターミナルを開き、SSH を使用して EC2 インスタンスに接続する。

```bash
ssh -i key.pem ec2-user@<パブリックIP>
```

---

## 3. ボリュームサイズの認識確認

### 3-1. デバイス構成の確認

ボリュームサイズが OS 上で認識されているかを確認する。

```bash
lsblk
```


- ディスクサイズが **AWS コンソールで変更したサイズ** になっていることを確認する  
- パーティションサイズが拡張されていない場合があるため注意する

---

## 4. パーティションの拡張（必要な場合）

### 4-1. パーティション有無の確認

`lsblk` の出力を確認し、以下を判別する。

- パーティションあり：`xvda1`、`nvme0n1p1` など  
- パーティションなし：`xvdf`、`nvme1n1` など  

※ パーティションが存在しない場合は、本章をスキップする

---

### 4-2. パーティションサイズの確認

```bash
lsblk
```

- パーティションサイズがボリュームサイズより小さい場合、拡張が必要

---

### 4-3. パーティションの拡張（growpart）

パーティションを拡張する。

```bash
sudo growpart /dev/xvda 1
```

- `/dev/xvda`：ディスク名  
- `1`：パーティション番号  
- 実際のデバイス名・番号は `lsblk` の結果に合わせて指定する

---

### 4-4. 拡張後の確認

```bash
sblk
```

- パーティションサイズがボリュームサイズと一致していることを確認する

---

## 5. ファイルシステムの拡張

### 5-1. ファイルシステムの確認

拡張前のファイルシステムサイズを確認する。

```bash
df -hT
```

---

### 5-2. ファイルシステムの拡張

使用しているファイルシステムに応じて、以下を実施する。

---

#### XFS ファイルシステムの場合

```bash
sudo xfs_growfs -d /
```

- `/` はマウントポイントを指定する  
- ルート以外の場合は対象のマウントポイントを指定する

---

#### ext4 ファイルシステムの場合

```bash
sudo resize2fs /dev/xvda1
```

- 対象のパーティションを指定する

---

### 5-3. 拡張後の確認

```bash
df -hT
```

- ファイルシステムサイズが拡張されていることを確認する

---

## 6. 注意事項
- EBS のサイズ変更だけでは、Linux のファイルシステムは自動で拡張されない  
- パーティションの有無により、必要な作業が異なる  
- ルートボリュームでも同様の手順で拡張可能  
- 作業前にスナップショットを取得しておくことが望ましい  

---

## 7. まとめ

1. EC2 インスタンスに接続する  
2. `lsblk` でボリュームサイズを確認する  
3. 必要に応じてパーティションを拡張する  
4. ファイルシステムを拡張する  
5. `df -hT` で拡張結果を確認する
