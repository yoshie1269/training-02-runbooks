# NFSサーバ構築 手順書（共有ディレクトリ設定・マウント確認）

---

## 概要

本手順書では、NFS（Network File System）を利用して  
サーバ間でディレクトリを共有する設定を行う。

本工程により、以下を実現する。

- NFS サーバ側で共有ディレクトリを作成する
- クライアント側で NFS マウントを設定する
- /etc/fstab により自動マウントを有効化する
- 共有領域の読み書きを確認する

---

## 前提条件

- NFS サーバがインストール・起動済みであること
- NFS クライアントが設定済みであること
- 両サーバが同一ネットワーク（172.31.0.0/16）に属していること
- root 権限で作業していること

---

## 1. root ユーザーに切り替える

---

### この工程でしていること
- システム設定や共有設定を行うために管理者権限へ切り替える

---

### 実行コマンド
```bash
sudo su -
```

---

### コマンドの意味
- sudo：管理者権限で実行
- su -：root 環境へ切り替え

---

### 確認方法とOKの目安
- プロンプトが `#` になること

---

## 2. 共有ディレクトリを作成する（サーバ側）

---

### この工程でしていること
- NFS で共有するためのディレクトリを作成する

---

### 実行コマンド
```bash
mkdir /share
```

---

### コマンドの意味
- mkdir：ディレクトリ作成
- /share：共有用ディレクトリ

---

### 確認方法とOKの目安
```bash
ls / | grep share
```

- /share が存在する

---

## 3. NFS 共有設定ファイルを編集する

---

### この工程でしていること
- どのディレクトリを
- どのネットワークに
- どの権限で共有するか設定する

---

### 実行コマンド
```bash
vi /etc/exports
```

---

### 記載内容
```bash
/share 172.31.0.0/16(rw,no_root_squash)
```

---

### 設定の意味

| 項目 | 内容 |
|------|------|
| /share | 共有対象ディレクトリ |
| 172.31.0.0/16 | 許可するネットワーク |
| rw | 読み書き許可 |
| no_root_squash | root権限を保持 |

---

### 注意点（重要）
- no_root_squash は学習用向け
- 本番ではセキュリティ上非推奨

---

### 確認方法とOKの目安
```bash
exportfs -v
```

- /share が表示される

---

## 4. マウント用ディレクトリを作成する（クライアント側）

---

### この工程でしていること
- NFS 共有を接続するための受け口を作成する

---

### 実行コマンド
```
mkdir /yakiniku
```

---

### コマンドの意味
- mkdir：ディレクトリ作成
- /yakiniku：マウントポイント

---

### 確認方法とOKの目安

```bash
ls / | grep yakiniku
```

---

## 5. fstab をバックアップする

---

### この工程でしていること
- 自動マウント設定前に復旧用バックアップを作成する

---

### 実行コマンド
```bash
cp /etc/fstab /etc/fstab.bak
```

---

### コマンドの意味
- cp：ファイルコピー
- .bak：バックアップ識別

---

### 確認方法とOKの目安
```bash
ls /etc | grep fstab
```

- fstab.bak が存在

---

## 6. 自動マウント設定を追加する

---

### この工程でしていること
- OS 起動時に自動で NFS をマウントする設定を行う

---

### 実行コマンド
```bash
vi /etc/fstab
```

---

### 追記内容
```bash
172.o.o.o:/share /yakiniku nfs4 defaults 0 0
```

---

### 各項目の意味

| 項目 | 内容 |
|------|------|
| 172.o.o.o:/share | NFSサーバの共有元 |
| /yakiniku | マウント先 |
| nfs4 | NFSバージョン |
| defaults | 標準オプション |
| 0 0 | fsck設定 |

---

### 注意点（重要）
- IP は NFS サーバのものに合わせる
- 記述ミスは起動失敗の原因になる

---

### 確認方法とOKの目安
- 記述にスペースズレがない

---

## 7. NFS をマウントする

---

### この工程でしていること
- 設定した共有領域を実際に接続する

---

### 実行コマンド
```bash
mount /yakiniku/
```

---

### コマンドの意味
- mount：ファイルシステム接続

---

### 確認方法とOKの目安
- エラーが出ない

---

## 8. マウント状態を確認する

---

### この工程でしていること
- NFS が正常に接続されているか確認する

---

### 実行コマンド
```bash
df
```

---

### コマンドの意味
- df：ディスク使用状況表示

---

### 確認方法とOKの目安
```bash
表示例：
172.o.o.o:/share ... /yakiniku
```

- /yakiniku が表示される

---

## 9. 共有ファイルの作成・確認

---

### この工程でしていること
- クライアント側から書き込み可能か確認する
- サーバ側と同期されているか検証する

---

### 実行コマンド
```bash
touch /yakiniku/test.txt
```
```bash
ls /yakiniku/
```
```bash
ls /share/
```

---

### コマンドの意味

| コマンド | 内容 |
|----------|------|
| touch | 空ファイル作成 |
| ls | ファイル一覧表示 |

---

### 確認方法とOKの目安

#### クライアント側
```bash
ls /yakiniku/
```

- test.txt が表示される

#### サーバ側
```bash
ls /share/
```

- test.txt が表示される

→ 同期成功

---

## 10. トラブル時の確認ポイント

---

### マウントできない場合

| 項目 | 確認内容 |
|------|----------|
| NFS状態 | systemctl status nfs-server |
| exports | exportfs -v |
| 通信 | ping NFSサーバ |
| ポート | netstat -ln \| grep 2049 |
| fstab | 記述ミス |

---

## 11. ここまでで完了していること

- /share が共有されている
- exports が正しく設定されている
- /yakiniku にマウントされている
- 自動マウントが設定されている
- ファイル同期が成功している

---

## 試験・研修向け重要ポイントまとめ

- /etc/exports は NFS の中核設定
- rw は読み書き可能
- no_root_squash は権限保持
- /etc/fstab で自動マウント
- df でマウント確認
- touch で同期確認
